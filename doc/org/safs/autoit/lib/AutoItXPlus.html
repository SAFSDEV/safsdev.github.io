Class AutoItXPlus

  • java.lang.Object
    • autoitx4java.AutoItX
      • org.safs.autoit.lib.AutoItXPlus

  • public class AutoItXPlus
extends autoitx4java.AutoItX
    AutoIt/AutoItX is the DLL/COM control, which can be used by different programming languages to access AutoIt through COM objects. Currently, 3rd part Java JAR autoitx4java has provided one java version AutoItX object by using JACOB to access this AutoIt/AutoItX COM object. It gives many common Java APIs to use AutoIt functionalities, but still lacks of APIs like "Ctrl + Click", "Alt + Click" to support SAFS keywords. Thus it requires creating SAFS version 'AutoItXPlus', which extends from 'autoitx4java/AutoItX', to access AutoIt functionalities through AutoIt/AutoItX COM object.

    Tao Xie

      • click

        public boolean click(java.lang.String title,
            java.lang.String text,
            java.lang.String controlID,
            java.lang.String button,
            int nClicks,
            java.awt.Point offset,
            java.lang.String specialKey)
        Workhorse of click action. It'll first use Java AWT Robot click call first, and then use AutoIt API to click if previous click action failed. Provide click action through command sending with. It can be assigned mouse button like 'left, middle, right', number of click times, click offset position and the special key holding when clicking.
        Parameters:
        title - String, title of the window to access.
        text - String, text of the window to access.
        controlID - String, control to interact with.
        button - String, button to click, "left", "right" or "middle".
        nClicks - int, number of times to click the mouse. It's supposed to be greater than or equal to 1.
        offset - Point, position to click within the control. It can be null or empty String. Default value is center.
        specialKey - String, keyboard key that be hold when click action happening. It can be null or empty String.
        Returns:
        boolean
