    • Constructor Detail

      • Variable

        public Variable(java.lang.String varname)
        Create a new Variable object with the given varname. Variable names cannot contain spaces, can contain alpha and numeric characters, and can contain a few special characters like '.'(period) and '_'(underscore).

        Note: Special characters should NOT be used where they may conflict with the separator used in your table records.

        SAFS DDVariable names are NOT case-sensitive, so variables named 'varname' and 'VARNAME' reference the same variable.

        Parameters:
        varname - valid DDVariable name (cannot be null or contain whitespace). The name provided will be trim()'d prior to use.\
        Throws:
        java.lang.IllegalArgumentException - if null or zero-length name is specified

        public java.lang.String getName()
        "varname" -- Returns the name of this variable with no leading variable identifier('^'). This is used at times when only the name of the variable is needed.

        Some SAFS commands want the names of variables.
        Attempting to provide the variable name with the variable identifier prefix will result in providing the variable's value instead of its name at runtime.

        Returns:
        varname
