Class SeleniumPlus.WDTimeOut

    public static class SeleniumPlus.WDTimeOut
extends java.lang.Object
     This class provides some static methods to set Selenium WebDriver's timeout thread-safely.

    History:

    Jul 8, 2014 (Lei Wang) Initial release.
    Jun 5, 2014 (Lei Wang) Add some non-thread safe method to set timeout for implicit-wait, page-load, and script.

      • setImplicitlyWait

        public static boolean setImplicitlyWait(long timeout,
                        java.util.concurrent.TimeUnit unit)
         Set the 'Web Driver's ImplicitlyWait timeout' and lock the 'timeout setting'.
 This method is thread-safe. If one thread has called this method,
 it will set a lock and the other threads will have to wait until the first thread
 unlock by calling resetImplicitlyWait(long, TimeUnit).
 The lock range is within the caller method.
 See the following example:
 
 try{
   setImplicitlyWait(timeout, unit);
   //doSomething();
 }catch(Exception e){
 }finally{
   resetImplicitlyWait(originalTimeout, originalUnit);
 }
        Parameters:
        timeout - long, the timeout to set
        unit - TimeUnit, the timeout's unit, seconds or milliseconds etc.
        Returns:
        boolean, true if the timeout has been set successfully.
        See Also:
        resetImplicitlyWait(long, TimeUnit)

      • setImplicitlyWait

        public static boolean setImplicitlyWait(long timeout,
                        java.util.concurrent.TimeUnit unit,
                        long waitLockTimeout)
        Set the 'Web Driver's ImplicitlyWait timeout'
        This method is thread-safe. To set 'timeout', thread needs to possess a lock. But if the lock is possessed by an other
        thread, this thread must wait. If this thread doesn't want to wait for the lock for a too
        time, this method provides a 'waitLockTimeout', after that timeout, this thread will quit.
        The lock range is within the caller method.
        Parameters:
        timeout - long, the timeout to set
        unit - TimeUnit, the timeout's unit, seconds or milliseconds etc.
        waitLockTimeout - long, the timeout to wait for obtaining the lock, in milliseconds
        Returns:
        boolean, true if the timeout has been set successfully.
        See Also:
        setImplicitlyWait(long, TimeUnit), resetImplicitlyWait(long, TimeUnit)
