      • verifyItemSelected

        protected void verifyItemSelected(Element element)
                           throws SeleniumPlusException
        As we call SAP API TreeNode.select() to show a node on page, which will set the 'selected' property to true. If we click that node again, this property 'selected' will be set to false, which will cause the verification fail. So we just test the value in parameter element, which is set before clicking the node.

      • showOnPage

        protected void showOnPage(Element element)
                   throws SeleniumPlusException
        In this implementation, we call node.select() to show the node on page. This API will also select the node of the tree, the 'selected' property will be set to true; but if we click the node again by mouse, this property 'selected' will be set to false!!! So we set the value of 'selected' to the element parameter, and in method verifyItemSelected(Element), if the element isSelected() then we will not try to get the value for the real SAP TreeNode object.
