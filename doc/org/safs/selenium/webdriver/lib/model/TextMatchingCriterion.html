org.safs.selenium.webdriver.lib.model

Class TextMatchingCriterion

  • java.lang.Object
    • org.safs.selenium.webdriver.lib.model.TextMatchingCriterion

  • public class TextMatchingCriterion
extends java.lang.Object
    This class contains text informations for searching an item in a list, tree, menu etc.
    1. The text/path to match
    2. If the match is partial of full, which means if the text is substring of the item's text in the list, tree etc.
    3. The expected matched index or expected matched indices.
      "expected matched index" is a single int value and it means the Nth matched node/path is wanted
      "expected matched indices" is an array of int value and it means the Nth matched node of each level in the path
      If matchIndexHierarchically() return true, then it contains an array; otherwise it just contains a single int value.

    History:

    Jun 26, 2014 (Lei Wang) Initial release.
    Oct 29, 2014 (Lei Wang) Add constructor with only one int value, to match only the index. Overload method matchText(): to match text at certain level for hierarchical structure.
